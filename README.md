# 情侣行程汇报小程序

一个用于情侣双方互相汇报行程并记录的小程序。

## 功能特点

- ✅ **双身份切换**：支持"我"和"TA"两个身份，方便情侣双方使用
- ✅ **行程汇报**：记录时间、地点和内容
- ✅ **行程列表**：按时间倒序显示所有行程记录
- ✅ **行程详情**：查看完整的行程信息
- ✅ **数据持久化**：使用本地存储保存数据

## 页面说明

### 1. 首页（行程列表）
- 显示所有行程记录，按时间倒序排列
- 顶部可切换用户身份（"我" / "TA"）
- 点击行程卡片可查看详情
- 不同身份用不同颜色标识

### 2. 添加行程页面
- 选择当前身份（"我" / "TA"）
- 选择时间（默认当前时间）
- 输入地点（必填）
- 输入内容描述（可选，最多200字）
- 提交后自动跳转到首页

### 3. 行程详情页面
- 显示完整的行程信息
- 可以删除该条记录

## 使用说明

1. **在微信开发者工具中打开项目**
   - 打开微信开发者工具
   - 选择"导入项目"
   - 选择本项目目录
   - 填写 AppID（可以使用测试号）

2. **配置 AppID**
   - 编辑 `project.config.json` 中的 `appid` 字段
   - 或使用测试号进行开发

3. **添加图标（可选）**
   - 在 `images/` 目录下添加以下图标：
     - `list.png` / `list-active.png`（行程列表图标）
     - `add.png` / `add-active.png`（添加行程图标）
   - 如果没有图标，小程序仍可正常运行

4. **自定义用户名称**
   - 编辑 `app.js` 中的 `userNames` 对象
   - 可以修改为你们的名字或昵称

## 技术说明

- **数据存储**：使用微信小程序的本地存储 API（wx.setStorageSync / wx.getStorageSync）
- **数据格式**：JSON 格式存储在本地
- **时间显示**：智能显示相对时间（刚刚、X分钟前、今天、昨天等）

## 注意事项

- 数据存储在本地，清除小程序数据会丢失所有记录
- 如需云端同步，可以后续接入微信云开发
- 建议定期备份重要数据
